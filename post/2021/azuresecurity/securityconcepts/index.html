Microsoft Identity Plattform - Create WEB API, SPA and Console Client

Sat, Feb 20, 2021 4-minute read

This series should describe you the concept and use of the Microsoft Identity Plattform.

I will use aspnetcore 5 webapi and implement authentication and authorization from scratch.

The most important know how ressource about the topic is this page.

What do we build?

  1. asp dotnetcore 5 webapi (REST) including authentication and autorization

  2. console client accessing the web-api (REST) with client secrets

  3. react SPA accassing the web-api (REST) using msal-browser and msal-react

What is Microsoft Identity Plattform?

To make a long story short, it is mostly Azure Actice Directory and some usefull libs added for the developer.

Step 1: Create the App Registration for the web-api (REST)

The first thing you have to do, is to create an App Registration. This will create a “container” which connects your app with the Microsoft Identity Plattform.

Only with the app registration you are able to implement client certificates, users and rolles etc.

public void Configure(IApplicationBuilder app, IWebHostEnvironment env)
{
    if (env.IsDevelopment())
    {
        app.UseDeveloperExceptionPage();
        app.UseSwagger();
        app.UseSwaggerUI(c => c.SwaggerEndpoint("/swagger/v1/swagger.json", "restapi v1"));
    }

    app.UseHttpsRedirection();

    app.UseRouting();

    app.UseAuthentication();

    app.UseAuthorization();

    app.UseEndpoints(endpoints =>
    {
        endpoints.MapControllers();
    });
}

public IEnumerable<WeatherForecast> Get()
{
    var rng = new Random();
    return Enumerable.Range(1, 5).Select(index => new WeatherForecast
    {
        Date = DateTime.Now.AddDays(index),
        TemperatureC = rng.Next(-20, 55),
        Summary = Summaries[rng.Next(Summaries.Length)]
    })
    .ToArray();
}
